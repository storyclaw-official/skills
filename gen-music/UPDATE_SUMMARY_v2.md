## 🎯 核心改进

### 1. 脚本功能增强 (kie_suno_api.py)

#### 新增功能

**跨平台下载支持：**
```python
✅ get_download_directory()     # 自动识别系统下载目录
✅ sanitize_filename()          # 清理文件名（移除非法字符）
✅ download_audio()             # 下载 MP3 文件到本地
```

**支持的操作系统：**
- ✅ macOS: `~/Downloads/`
- ✅ Windows: `%USERPROFILE%\Downloads\`
- ✅ Linux: `~/Downloads/` 或 `$XDG_DOWNLOAD_DIR`

**新增命令行参数：**
```bash
--download              # 自动下载 MP3 文件
--download-dir PATH     # 指定自定义下载目录
```

**增强的输出：**
```json
{
  "title": "Summer Vibes",
  "prompt": "一首关于夏天的欢快流行歌",
  "audioUrl": "https://assets.giggle.pro/audio/xxx.mp3",
  "tags": "pop, upbeat, summer",
  "localPath": "/Users/username/Downloads/Summer_Vibes.mp3"  // 新增
}
```

### 3. 渐进式披露实施

#### v1.2 问题
```
用户："生成音乐"
→ 一次性询问：歌词？风格？标题？人声？
❌ 信息过载，用户困惑
```

#### v2.0 改进
```
用户："生成音乐"
    ↓
步骤 1: 智能分析输入 ✅ 避免不必要询问
    ↓
步骤 2: 询问音乐类型（仅在需要时）✅ 简单选择
    ↓
步骤 3: 分步收集信息 ✅ 逐步展开
    ↓
步骤 4: 展示结果 ✅ 清晰呈现
    ↓
步骤 5: 询问是否下载 ✅ 可选操作
```

**优势：**
- ✅ 每步只关注一个决策
- ✅ 提供时间预期（60-90秒 / 90-120秒）
- ✅ 推荐选项明确标注"(推荐)"
- ✅ 每个选项都有详细说明

### 4. 用户体验优化

#### 智能分析

**v1.2:** 总是询问
```
用户："生成一首关于夏天的音乐"
→ 还是会询问"您是否有歌词？"
❌ 浪费时间
```

**v2.0:** 智能判断
```
用户："生成一首关于夏天的音乐"
→ 识别到描述，直接执行简化模式
✅ 节省 2-3 轮对话
```

#### 简化模式优先

**推荐话术（新增）：**
```
"我们推荐使用简化模式：
✅ 只需一句话描述
✅ AI 自动生成歌词和音乐
✅ 更快、更简单（~60-90秒）

您只需告诉我想要什么样的音乐即可！"
```

#### 时间预期（新增）

| 模式 | 生成时间 | 说明 |
|-----|---------|------|
| 简化模式 - AI 创作 | ~60-90秒 | 快速创作 |
| 自定义模式 - 精确控制 | ~90-120秒 | 更多参数 |
| 简化模式 - 纯音乐 | ~60-90秒 | 无歌词生成 |

### 5. 下载功能优化

#### v1.2: 仅保存文本
```
问题: "是否保存输出内容？"
→ 保存到 ~/Downloads/歌曲名.txt
❌ 用户还需要手动下载 MP3
```

#### v2.0: 直接下载 MP3
```
问题: "是否将音乐文件下载到本地？"
→ 自动下载 MP3 到 ~/Downloads/歌曲名.mp3
✅ 一步到位
```

**跨平台支持：**
```python
# 自动识别操作系统
if platform.system() == "Darwin":      # macOS
    → ~/Downloads/
elif platform.system() == "Windows":   # Windows
    → %USERPROFILE%\Downloads\
else:                                  # Linux
    → ~/Downloads/ 或 $XDG_DOWNLOAD_DIR
```

**文件名处理：**
```python
✅ 移除非法字符: < > : " / \ | ? *
✅ 自动去重: 歌曲名.mp3 → 歌曲名_1.mp3
✅ 长度限制: 最多 200 字符
```
