## 提示词基础

### 什么是好的提示词？

**✅ 好的提示词特征:**
- 清晰具体的主体描述
- 明确的风格指定
- 适当的细节补充
- 简洁但不失完整

**❌ 不好的提示词特征:**
- 过于简短（"猫"）
- 过于冗长（重复描述）
- 模糊不清（"好看的图片"）
- 自相矛盾（"白色的黑猫"）

---

### 提示词长度建议

| 长度 | 适用场景 | 示例 |
|-----|---------|------|
| **10-20 词** | 简单主体 | "一只橘色的猫咪坐在窗边" |
| **20-50 词** | 日常使用（推荐）| "一只橘色的猫咪坐在窗边，阳光从窗外洒进来，温馨氛围，水彩画风格" |
| **50-100 词** | 复杂场景 | "未来主义城市景观，高楼大厦，霓虹灯牌，赛博朋克风格，夜景，雨后的街道反射着五彩灯光，细节丰富，高动态范围" |
| **>100 词** | 极度精细控制 | 仅在必要时使用，注意不要超过 20000 字符限制 |

## 提示词结构

### 推荐结构模板

```
[主体] + [动作/状态] + [环境/背景] + [氛围/情感] + [艺术风格] + [技术细节]
```

**示例:**
```
主体: 一只橘色的猫咪
动作/状态: 坐在窗边
环境/背景: 窗外是美丽的花园
氛围/情感: 温馨宁静的午后
艺术风格: 水彩画风格
技术细节: 柔和的光线，高细节
```

**完整提示词:**
```
一只橘色的猫咪坐在窗边,窗外是美丽的花园,温馨宁静的午后,水彩画风格,柔和的光线,高细节
```

---

### 简化结构（快速使用）

```
[主体] + [关键特征] + [风格]
```

**示例:**
```
猫咪坐在窗边,阳光洒进,水彩画风格
```

## 图生图提示词

### 风格转换

**目标:** 保持内容，改变风格

**模板:**
```
转为[目标风格],保持[要保留的元素],[额外要求]
```

**示例:**
```bash
--prompt "转为油画风格,保持主体和构图不变,增加笔触质感" \
--image-input "https://example.com/photo.jpg"
```

---

### 图像优化

**目标:** 提升质量和细节

**模板:**
```
提升[质量方面],增强[细节方面],[保持原有特征]
```

**示例:**
```bash
--prompt "提升清晰度和细节,增强色彩饱和度,保持自然真实感" \
--image-input "https://example.com/low-res.jpg"
```

---

### 背景替换

**目标:** 保留主体，更换背景

**模板:**
```
保持主体,替换背景为[新背景描述]
```

**示例:**
```bash
--prompt "保持人物主体,替换背景为梦幻星空,添加柔和光晕效果" \
--image-input "https://example.com/portrait.jpg"
```

## 多图融合提示词

### 风格混合

**目标:** 融合多种艺术风格

**模板:**
```
融合这些图像的[特定方面],创造[期望效果],[技术要求]
```

**示例:**
```bash
--prompt "融合这些图像的色彩和构图,创造独特的视觉冲击,保持和谐统一" \
--image-input "url1" "url2" "url3"
```

---

### 元素组合

**目标:** 提取并组合不同图像的元素

**模板:**
```
提取并组合[元素列表],创造[场景描述],[风格要求]
```

**示例:**
```bash
--prompt "提取图1的建筑元素和图2的自然元素,创造和谐共生的生态城市,未来主义风格" \
--image-input "urban.jpg" "nature.jpg"
```

**示例:**

## 提示词优化技巧

### 1. 从简到繁

**迭代过程:**

**第一版（简单）:**
```
猫咪
```

**第二版（添加细节）:**
```
橘色猫咪坐在窗边
```

**第三版（添加氛围和风格）:**
```
橘色猫咪坐在窗边,阳光洒进,温馨氛围,水彩画风格
```

**第四版（优化完善）:**
```
橘色猫咪安静地坐在木质窗台上,温暖的午后阳光透过窗户洒进来,营造出宁静温馨的氛围,水彩画风格,柔和色调,高细节
```

---

### 2. 使用权重和强调（部分模型支持）

**语法:**
```
(关键词:权重)
```

**示例:**
```
(橘色猫咪:1.2), 窗边, (柔和光线:1.1), 水彩画风格
```

**注意:** Nano Banana Pro 的权重支持需要测试验证

---

### 3. 负面提示词（指定不想要的元素）

**示例:**
```
美丽的风景, 湖泊, 山脉, 不要有人物, 不要有建筑
```

---

### 4. 参考艺术家风格（谨慎使用）

**示例:**
```
山水画, 宋代山水风格, 云雾缭绕
```

**注意:** 避免直接使用在世艺术家的名字

---

### 5. 使用具体数字

**示例:**
```
4K 超高清, 8K 分辨率, 35mm 镜头, f/1.8 光圈, ISO 100
```

**效果:** 增加专业感和技术规格暗示

---

### 6. 组合多个风格

**示例:**
```
赛博朋克 + 水墨画风格, 东西方美学融合, 霓虹灯与墨色交织
```

## 避免的常见错误

### ❌ 错误 1: 过于简短

**错误示例:**
```
猫
```

**问题:** 缺乏足够信息，结果不可预测

**改进:**
```
一只橘色的猫咪坐在窗边,阳光洒进,温馨氛围,水彩画风格
```

---

### ❌ 错误 2: 自相矛盾

**错误示例:**
```
白色的黑猫, 白天的夜景, 写实的抽象画
```

**问题:** 矛盾的描述导致模型困惑

**改进:**
```
黑色的猫咪, 夜景, 写实风格的绘画
```

---

### ❌ 错误 3: 过度堆砌关键词

**错误示例:**
```
猫,窗户,阳光,温馨,水彩,柔和,细节,高清,4K,8K,超清晰,完美,最佳质量,杰作...
```

**问题:** 关键词过多反而降低质量

**改进:**
```
橘色猫咪坐在窗边,阳光洒进,温馨氛围,水彩画风格,高细节
```

---

### ❌ 错误 4: 模糊不清

**错误示例:**
```
一张好看的图片, 漂亮的风景, 酷炫的效果
```

**问题:** "好看"、"漂亮"、"酷炫" 过于主观

**改进:**
```
雪山湖泊,日出时分,金色光线,倒影清晰,宁静氛围,风景摄影
```

---

### ❌ 错误 5: 过度冗长

**错误示例:**
```
一只非常非常可爱的橘色小猫咪静静地坐在木质的窗台上面,窗外是非常美丽的花园... (重复描述 500 字)
```

**问题:** 冗余信息不会提升质量，反而浪费配额

**改进:**
```
橘色小猫坐在木质窗台,窗外花园景色,阳光明媚,温馨宁静,水彩画风格
```

---

### ❌ 错误 6: 忽略风格

**错误示例:**
```
一只猫坐在窗边
```

**问题:** 未指定风格，结果可能不符合预期

**改进:**
```
一只猫坐在窗边,水彩画风格
```
或
```
一只猫坐在窗边,写实摄影风格
```

## 总结

### 黄金法则

1. **清晰 > 复杂**: 简洁明了的描述优于复杂堆砌
2. **具体 > 抽象**: 具体描述优于抽象概念
3. **结构化思考**: 使用结构化模板组织提示词
4. **迭代优化**: 从简单开始，逐步优化
5. **测试验证**: 使用 1K 快速测试，满意后生成高清

### 快速检查清单

✅ 是否包含清晰的主体？
✅ 是否指定了风格？
✅ 是否有适当的细节描述？
✅ 是否避免了自相矛盾？
✅ 是否长度适中（20-50词）？
✅ 是否可以更具体一些？
